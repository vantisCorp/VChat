## 📋 Dual-Licensing Model

V-COMM uses a **dual-licensing model** that balances open-source freedom with sustainable development funding.

### 1. GNU Affero General Public License v3.0 (AGPLv3)

The default license for V-COMM is AGPLv3, which ensures:
- ✅ Free to use for open-source projects
- ✅ Free to modify and distribute
- ✅ Copyleft protection - derivatives must be open-source
- ✅ Network clause - SaaS usage requires source disclosure
- ❌ Cannot use in proprietary software without commercial license

**When AGPLv3 applies:**
- Personal projects
- Open-source projects
- Academic research
- Non-profit organizations
- Any project willing to share source code

### 2. Commercial License

For organizations that cannot comply with AGPLv3 requirements, we offer commercial licenses:

**Commercial License Benefits:**
- ✅ Use V-COMM in proprietary software
- ✅ No obligation to disclose source code
- ✅ Priority support
- ✅ Custom feature development
- ✅ SLA guarantees
- ✅ Indemnification

**When Commercial License is required:**
- SaaS products using V-COMM
- Proprietary enterprise software
- Government classified systems
- Any product not disclosing source code

## ❓ Frequently Asked Questions

<details>
<summary><strong>Do I need a commercial license for my startup?</strong></summary>

If your startup's software is open-source (released under a compatible license), you can use AGPLv3. If your software is proprietary or you're building a SaaS without disclosing source code, you need a commercial license.

**Startup-friendly option:** We offer discounted rates for early-stage companies.
</details>

<details>
<summary><strong>Can I modify V-COMM under AGPLv3?</strong></summary>

Yes! AGPLv3 allows modification, but you must:
1. Clearly indicate your modifications
2. Release your modified version under AGPLv3
3. Make source code available to users (including over network)
</details>

<details>
<summary><strong>What if I contribute code to V-COMM?</strong></summary>

All contributors must sign our Contributor License Agreement (CLA). The CLA ensures:
- You retain copyright to your contributions
- V-COMM can use your code under both AGPLv3 and commercial licenses
- Your contributions are properly attributed

See [CONTRIBUTING.md](CONTRIBUTING.md) for details.
</details>

<details>
<summary><strong>Can I use V-COMM internally at my company?</strong></summary>

Yes! Internal use (not distributed externally) is allowed under AGPLv3. However, if you:
- Distribute the software externally
- Provide it as a service to external users
- Modify it without sharing changes

...you may need a commercial license.
</details>

<details>
<summary><strong>What happens if I violate AGPLv3?</strong></summary>

AGPLv3 violations can result in:
- License termination
- Legal action requiring compliance
- Damages in some jurisdictions

We prefer working with users to achieve compliance. Contact us if you have questions.
</details>
